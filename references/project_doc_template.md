# 项目文档模板

项目文档是对项目本身的客观描述，随着学习深入逐步完善。与个人学习笔记不同，项目文档不包含个人感受和学习过程，只记录关于项目的事实性知识。可分享给团队成员阅读。

项目文档存放在 `learning-notes/project-doc/` 目录下，按关注点拆分为以下文件。

---

| 数据库 | PostgreSQL | |

- 测试：`go test ./...`

## 核心流程（`flows.md`）

每次做执行链路分析后，将流程的客观描述追加到此文件。

```markdown
# 🔄 核心流程

> 最后更新：YYYY-MM-DD

## [流程名称]（如：用户注册流程）

### 触发方式
POST /api/users

### 流程图
(Mermaid sequenceDiagram)

### 关键步骤
1. Handler 接收请求，绑定参数并校验
2. Service 执行业务规则校验
3. Service 将 DTO 转换为 Entity
4. Repository 持久化到数据库
5. Service 将 Entity 转换为 VO 返回

### 数据变换
| 阶段 | 数据类型 | 说明 |
|------|----------|------|
| 入参 | CreateUserRequest (DTO) | 客户端提交的数据 |
| 业务层 | User (Entity) | 内部业务实体 |
| 返回 | UserVO (VO) | 返回给客户端的视图 |

### 错误处理
| 错误场景 | 处理方式 | HTTP 状态码 |
|----------|----------|-------------|
| 参数校验失败 | 返回错误详情 | 400 |
| 业务规则冲突 | 返回提示信息 | 409 |
| 内部错误 | 返回通用错误 | 500 |

---

## [下一个流程]
...
```
